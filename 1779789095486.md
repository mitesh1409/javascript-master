# Closure

In JavaScript, a closure is a function that "remembers" and continues to access variables from its outer scope even after that outer function has finished executing.

**How Closures Work**  
Closures are created every time a function is defined.  
They rely on "lexical scoping", which means the scope of a function is determined  
by its position in the source code.  

When you nest a function inside another, the inner function has access to:  
* Its own local variables.
* Variables in the outer function’s scope.
* Global variables.

Even if the outer function returns and its execution context is removed from the stack, the inner function maintains a reference to that outer scope (its "lexical environment") in memory.

Example  

```javascript
function outer() {
  let counter = 0; // Variable in the outer scope

  return function inner() {
    counter++; // Accesses 'counter' from outer scope
    console.log(counter);
  };
}

const myClosure = outer(); 
myClosure(); // 1
myClosure(); // 2 (it remembers the value of counter!)
```

**Common Use Cases**  

1. **Data Encapsulation (Private Variables):**  
   You can hide variables from the global scope, making them accessible only through specific functions.
2. **Function Factories:**  
   Creating multiple functions with different "pre-set" data (e.g., a function that adds 5 and another that adds 10).
3. **Callbacks and Async Code:**  
   Closures allow event handlers, setTimeout, or Promises to remember variables from the time they were created.
4. **Currying:**  
   Transforming a function that takes multiple arguments into a series of functions that take one argument at a time.

**Things to Watch Out For**  

**Memory Leaks:**  
Because closures keep references to outer variables, those variables aren't garbage collected as long as the closure exists. Excessive use can lead to higher memory consumption.

**Performance:**  
Overusing closures in performance-critical loops can sometimes lead to overhead.
